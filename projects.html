Linux System Hardening Project

Context: A Linux system was audited and found to have weak authentication policies, exposed services, and insufficient access controls.

Risk Identified: Unauthorized access, privilege escalation, and non-compliance with security best practices.

Tools Used: Lynis, SSH, UFW, PAM, systemctl

Actions Taken:

  • Performed a security audit using Lynis
  • Hardened SSH configuration
  • Enforced strong password and account policies
  • Configured firewall rules to reduce attack surface

Outcome: Improved system security posture and alignment with CIS Benchmarks.
